Git bad commit

Что делать, если сделал правки а pull не сделал, а измениня уже есть в origin

# 0. Отменяем merge через терминал vscode
git merge --abort

# 1. Выносим изменения в отдельную ветку
git checkout -b <название-новой-ветки>

# 2. Переключаемся на векту которую испортили
git checkout <название-оригинальной-ветки>

# 3. Смотрим лог чтобы найти предпоследний коммит, когда ещё всё было хорошо
git log

# 4. Отменяем коммиты до состояния когда всё было хорошо
git reset --hard 

# 5. Тянем изменения из origin
git pull

# 6. Делаем слияние с веткой в которой новые правки (пункт 1)
git merge <название-новой-ветки>

# 7. Решаем проблемы merge в vscode и подтверждаем их коммитом
git commit -m "Merge..."

# 8. Пушим измениня в оригинальную ветку
git push
